Interesting:
  • Note how anything wrapped in ios scroll css (-webkit-overflow-scrolling: touch;) dissapears if you scroll down a bit, this is the bug.
  • Anything using -webkit-overflow-scrolling: touch; looks a bit blurry. Im guessing because it is hardware accelerated.
  • E. The text wrapped in a div does allow single finger scrolling with "overflow:scroll;". But this does NOT work for an IFrame or Object in a div. I guess because the touch is going to the contained document, not the parent.
  • Object appears to offer no benefit over IFrame
